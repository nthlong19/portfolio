Crypto Currency Project

Business Model


Enterprise K is a cryptocurrency wallet company with millions of users worldwide. Each user can have one or multiple wallets, and within each wallet, there can be one or multiple types of cryptocurrencies. When users register on the K platform, they need to import one or several of their wallets onto the platform. Since it operates on blockchain technology, users do not need to declare their identity. However, information about the device they use to log in is recorded, including the type of device, platform (web/mobile), operating system, IP address (geographical location of the IP can be collected) and so on.

The K platform serves as a playground for users to utilize their wallets to exchange the cryptocurrencies they hold (swap). Users swap cryptocurrencies because they believe that the cryptocurrencies they exchange will appreciate in value in the future. There are many types of cryptocurrencies, but they are categorized into two main types: stablecoins and altcoins. Stablecoins are cryptocurrencies pegged to real-world currencies like USD, YEN, RMB, or gold, whereas altcoins are cryptocurrencies that float freely in the market, such as Bitcoin.

For each transaction, customers exchange one type of cryptocurrency for another based on its temporary value. For instance, if cryptocurrency A is priced at $1 per unit at that time, a customer might exchange it for 2 units of cryptocurrency B priced at $0.5 per unit. The K platform charges a transaction fee of 2%, which is paid in the exchanged cryptocurrency (cryptocurrency B in this example).

2. Create dim_wallets table

WITH wallet_address_row_number AS(
            SELECT dbo.fact_swap.*
            , ROW_NUMBER() OVER (PARTITION BY wallet_address ORDER BY country_code) AS sq_num
            FROM dbo.fact_swap
            )

SELECT wallet_address, os, platform, country_code 
            INTO dim_wallets
            FROM wallet_address_row_number
            WHERE sq_num = 1
            ORDER BY wallet_address

3. Create dim_date table


SELECT DISTINCT date AS datekey,
                day(date) as day,

                       WHEN datepart(dw,date) = 7 THEN 'Sunday'

4. Create fact_price table


WITH source_token AS(
        SELECT DISTINCT source_token AS token_id, date AS datekey, source_price AS price
        FROM dbo.fact_swap
        ), 
dest_token AS(
        SELECT DISTINCT dest_token AS token_id, date AS datekey, dest_price AS price
        FROM dbo.fact_swap
        ),
union_token AS(
        SELECT DISTINCT token_id, 
                        datekey, 
                        price 
        FROM source_token
        UNION
        SELECT DISTINCT token_id, 
                        datekey, 
                        price 
        FROM dest_token
        ),
union_token_row_number AS(
        SELECT union_token.*, 
               ROW_NUMBER() OVER (PARTITION BY token_id, datekey ORDER BY price) AS seq_num
        FROM union_token
        )
SELECT token_id, 
       datekey, 
       price
INTO fact_price
FROM union_token_row_number
WHERE seq_num = 1
